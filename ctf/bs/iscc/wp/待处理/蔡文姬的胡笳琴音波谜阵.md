# 蔡文姬的胡笳琴音波谜阵 Writeup

## 题目信息
- **题目类型**: Mobile Reverse Engineering (Android)
- **附件**: `app-release.apk` (约 103MB)

## 题目描述
蔡文姬的胡笳琴中隐藏着失传已久的秘宝。琴案之上，八位不同的幻影（Skin）对应着宫、商、角、徵、羽等古乐音阶。唯有按照特定的旋律顺序拨动琴弦，才能引发胡笳琴内"焦尾残卷"的共鸣。

然而，音波之中暗藏杀机。琴音经过洗礼化作流动的密钥，激活了古老的禁阵。更有一层幻象笼罩在最终的秘宝残影之上。如果你试图用凡间的窥视之术去偷看琴音的真相，幻阵的方位将会悄然偏移。

## 解题过程

### 1. APK 结构分析

APK 体量 103MB，包含 Flutter/Unity/native 资源。关键文件：
- `lib/` 下的 `libmobile04.so` — 核心 native 校验逻辑
- `assets/appview_cache.db` — SQLCipher 加密数据库
- `classes.dex` — Java 层控制流

### 2. Java 层分析（Smali 反编译）

使用 `apktool` 解包 APK，定位到 `com/example/mobile04` 包。

**MainActivity** 中限定了输入格式：
- 必须为 `ISCC{...}`，内部 32 字符送 native 校验
- 用户需要按顺序点击 8 个图标按钮，形成一段"旋律序列"

**八个按钮对应的序列码**：
| 按钮编号 | 序列码 |
|----------|--------|
| image1   | DANCE  |
| image2   | CWJ    |
| image3   | ROSE   |
| image4   | GOAL   |
| image5   | MAGI   |
| image6   | FLOW   |
| image7   | STAR   |
| image8   | XMAS   |

点击序列首先通过 `Z1.l(path, cw, inner)` 进行校验，其中 `cw` 由序列码经过以下变换生成：
1. `bit_swap(sequence)` — 按位交换
2. `FNV1a64` 哈希
3. `TEA` 加密（密钥由 `d(int)` 函数从 SO 常量区派生）

### 3. Native 层分析（libmobile04.so）

使用 `llvm-objdump` / `llvm-readobj` 定位关键函数：

- **`_n_impl`**: 主校验入口，参数为 `(path, cw, inner)`
- **`_f4`**: Base58 解码
- **`_f3`**: RC6-32/20 块加密/解密

**Native 校验流程**：

1. 取 `cache.dat` 前 16 字节作为盐值
2. `CRC32(cw || salt16)` 作为滚动 XOR key，解密数据库后半段
3. 从解密数据中提取 `seed1` (offset 0x80) 和 `seed2` (offset 0xb4) 作为 RC6 种子
4. 以 `"CaiWenJi_16_Key!"` 作为 RC6 key 进行密钥扩展
5. 从 `.rodata` 取 Base58 目标串：`CTmF27qJYS8YF964PXoSQLGieZd45LKw5CFfkpMnSmvN`
6. Base58 解码后与 5 字节 XOR mask `"Hujia"` 异或得到 RC6 密文
7. RC6 解密后与用户输入的 inner (32 chars) 比较

**常量提取**：
- `d_consts`: 位于 SO 偏移 `0x3dbb0` 处的 4 个 uint32
- XOR key: `Hujia`
- Static RC6 key: `CaiWenJi_16_Key!`
- Base58 target: `CTmF27qJYS8YF964PXoSQLGieZd45LKw5CFfkpMnSmvN`

### 4. 穷举破解

8 个序列码的全排列仅有 8! = 40320 种，可离线穷举：

```python
import itertools, struct, zlib, string

def u32(x): return x & 0xffffffff
def rol32(x, n): return u32((x << (n & 31)) | (x >> (32 - (n & 31))))
def ror32(x, n): return u32((x >> (n & 31)) | (x << (32 - (n & 31))))

def fnv1a64(bs):
    h = 0xcbf29ce484222325
    for b in bs:
        h ^= b
        h = (h * 0x100000001b3) & 0xffffffffffffffff
    return h

def tea_encrypt(v0, v1, key):
    delta = 0x9e3779b9
    s = 0
    for _ in range(32):
        s = u32(s + delta)
        v0 = u32(v0 + ((u32(v1 << 4) + key[0]) ^ u32(v1 + s) ^ (u32(v1 >> 5) + key[1])))
        v1 = u32(v1 + ((u32(v0 << 4) + key[2]) ^ u32(v0 + s) ^ (u32(v0 >> 5) + key[3])))
    return v0, v1

def b58decode(s):
    alphabet = '123456789ABCDEFGHJKLMNPQRSTUVWXYZabcdefghijkmnopqrstuvwxyz'
    num = 0
    for ch in s: num = num * 58 + alphabet.index(ch)
    out = num.to_bytes((num.bit_length() + 7) // 8, 'big') if num else b''
    return b'\x00' * (len(s) - len(s.lstrip('1'))) + out

codes = ['FLOW','CWJ','ROSE','XMAS','STAR','GOAL','MAGI','DANCE']
```

## 总结

本题为 Android 移动逆向工程题，核心考点：
1. **APK 逆向** — smali 代码阅读、native 函数定位
2. **自定义加密链** — FNV64 → TEA → CRC32 → RC6 的混合加密
3. **Base58 编码** — 类似 Bitcoin base58check
4. **离线穷举** — 利用 8! 可解空间暴力搜索
